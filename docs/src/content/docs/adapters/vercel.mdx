This adapter allows you to deploy your Vite project to [Vercel](https://vercel.com/). Learn how in our [Vercel deployment guide](/deploy/vercel/).

Vercel is a deployment platform that allows you to host your site by connecting directly to your GitHub repository. This adapter builds your Vite project for deployment through Vercel.

### Manual install

<Steps>

1. Add `@vite-deploy/vercel` to your project's dependencies using your preferred package manager:

    <PackageManagerTabs>
      <Fragment slot="npm">
        ```sh
        npm install @vite-deploy/vercel
        ```
      </Fragment>
      <Fragment slot="pnpm">
        ```sh
        pnpm add @vite-deploy/vercel
        ```
      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn add @vite-deploy/vercel
        ```
      </Fragment>
    </PackageManagerTabs>

2. Add the plugin to your Vite config file:

    ```js title="vite.config.js" ins={1,5-8}
    import vercel from "@vite-deploy/vercel";
    import { defineConfig } from "vite";

    export default defineConfig({
      plugins: [vercel({
        output: "server",
        handlerEntrypoint: "./src/handler.ts",
      })]
    });
    ```

3. Update your TypeScript config file to include necessary types:

    ```jsonc title="tsconfig.json" ins={5}
    {
      "compilerOptions": {
        "types": [
          "vite/client",
          "@vite-deploy/vercel/types"
        ]
      },
    }
    ```

4. Create the handler entrypoint:

    ```ts title="src/handler.ts"
    import type { ExportedHandler } from "@vite-deploy/vercel";

    export default {
      fetch(request) {
        const url = new URL(request.url);
        return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
      },
    } satisfies ExportedHandler;
    ```

5. Start the development server:

    <PackageManagerTabs>
      <Fragment slot="npm">
        ```sh
        npm run dev
        ```
      </Fragment>
      <Fragment slot="pnpm">
        ```sh
        pnpm dev
        ```
      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn dev
        ```
      </Fragment>
    </PackageManagerTabs>

</Steps>

## Options

The Vercel adapter accepts [output related options](/outputs/). It also accepts the following:

### `handlerEntrypoint`

<p>**Type:** `string | URL`</p>

Specifies what module should be used. It accepts:

- Paths relative to Vite's root: `./src/handler.ts`.
- Absolute paths: `/foo/handler.ts`.
- Package specifiers: `@my-pkg/handler`.
- URLs: `new URL("./src/handler.ts", import.meta.url)`.

```js title="vite.config.js" {7}
import vercel from "@vite-deploy/vercel";
import { defineConfig } from "vite";

export default defineConfig({
    plugins: [vercel({
        // ...
        handlerEntrypoint: "./src/handler.ts",
    })]
});
```

The module must return a `ExportedHandler`, which handles requests:

```ts title="src/handler.ts"
import type { ExportedHandler } from "@vite-deploy/vercel";

export default {
  fetch(request) {
    const url = new URL(request.url);
    return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
  },
} satisfies ExportedHandler;
```

### `requestLoggingLevel`

<p>
**Type:** `"silent" | "info" | undefined`<br />
**Default:** `"info"`
</p>

Specifies if requests are logged in the terminal. It can be disabled if you already handle logging at runtime:

```js title="vite.config.js" {7}
import vercel from "@vite-deploy/vercel";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [vercel({
    // ...
    requestLoggingLevel: "silent"
  })]
});
```

## Static assets

Static assets work out of the box. The adapter injects a header to cache assets under [`build.assetsDir`](https://vite.dev/config/build-options#build-assetsdir).
