This adapter allows you to deploy your Vite project to [Netlfy](https://www.netlify.com/). Learn how in our [Netlify deployment guide](/deploy/netlify/).

Netlify is a deployment platform that allows you to host your site by connecting directly to your GitHub repository. This adapter builds your Vite project for deployment through Netlify.

<Steps>

      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn add @vite-deploy/netlify
        ```
      </Fragment>
    </PackageManagerTabs>

2. Add the plugin to your Vite config file:

    ```js title="vite.config.js" ins={1,5-8}
    import netlify from "@vite-deploy/netlify";
    import { defineConfig } from "vite";

    export default defineConfig({
      plugins: [netlify({
        output: "server",
        handlerEntrypoint: "./src/handler.ts",
      })]
    });
    ```

3. Update your TypeScript config file to include necessary types:

    ```jsonc title="tsconfig.json" ins={5}
    {
      "compilerOptions": {
        "types": [
          "vite/client",
          "@vite-deploy/netlify/types"
        ]
      },
    }
    ```

4. Create the handler entrypoint:

    ```ts title="src/handler.ts"
    import type { ExportedHandler } from "@vite-deploy/netlify";

    export default {
      fetch(request) {
        const url = new URL(request.url);
        return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
      },
    } satisfies ExportedHandler;
    ```

5. Start the development server:

      </Fragment>

</Steps>

## Options

The Netlify adapter accepts [output related options](/outputs/). It also accepts the following:

### `handlerEntrypoint`

<p>**Type:** `string | URL`</p>

Specifies what module should be used. It accepts:

- Paths relative to Vite's root: `./src/handler.ts`.
- Absolute paths: `/foo/handler.ts`.
- Package specifiers: `@my-pkg/handler`.
- URLs: `new URL("./src/handler.ts", import.meta.url)`.

```js title="vite.config.js" {7}
import netlify from "@vite-deploy/netlify";
import { defineConfig } from "vite";

export default defineConfig({
    plugins: [netlify({
        // ...
        handlerEntrypoint: "./src/handler.ts",
    })]
});
```

The module must return a `ExportedHandler`, which handles requests:

```ts title="src/handler.ts"
import type { ExportedHandler } from "@vite-deploy/netlify";

export default {
  fetch(request, context) {
    const url = new URL(request.url);
    return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
  },
} satisfies ExportedHandler;
```

### `config`

<p>
**Type:** `NetlifyPluginOptions | undefined`<br />
**Default:** `undefined`
</p>

Options forwarded to [`@netlify/vite-plugin`](https://www.npmjs.com/package/@netlify/vite-plugin), except `middleware` and `build`:

```js title="vite.config.js" {7-9}
import netlify from "@vite-deploy/netlify";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [netlify({
    // ...
    config: {
      blobs: { enabled: true },
    }
  })]
});
```

### `requestLoggingLevel`

<p>
**Type:** `"silent" | "info" | undefined`<br />
**Default:** `"info"`
</p>

Specifies if requests are logged in the terminal. It can be disabled if you already handle logging at runtime:

```js title="vite.config.js" {7}
import netlify from "@vite-deploy/netlify";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [netlify({
    // ...
    requestLoggingLevel: "silent"
  })]
});
```

## Context

You can access [Netlify-specific context](https://docs.netlify.com/build/functions/api/#netlify-specific-context-object) in 2 ways:

- From the 2nd argument of [`handlerEntrypoint`](#handlerentrypoint)'s fetch handler:

  ```ts title="src/handler.ts" "context"
  import type { ExportedHandler } from "@vite-deploy/netlify";

  export default {
    fetch(request, context) {
      const url = new URL(request.url);
      return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
    },
  } satisfies ExportedHandler;
  ```

- From the [`Netlify` global](https://docs.netlify.com/build/functions/api/#netlify-global-object):

  ```ts title="src/handler.ts" "Netlify.context"
  import type { ExportedHandler } from "@vite-deploy/netlify";

  export default {
    fetch(request) {
      Netlify.context;
      const url = new URL(request.url);
      return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
    },
  } satisfies ExportedHandler;
  ```

  `Netlify.context` is only set within request scope, otherwise it is `null`.

When not on Netlify nor used through the [Netlify CLI](https://docs.netlify.com/api-and-cli-guides/cli-guides/get-started-with-cli/), context properties are emulated.

## Static assets

Static assets work out of the box. The adapter injects a header to cache assets under [`build.assetsDir`](https://vite.dev/config/build-options#build-assetsdir).
