This adapter allows you to deploy your Vite project to [Cloudflare](https://www.cloudflare.com/). Learn how in our [Cloudflare Workers deployment guide](/deploy/cloudflare-workers/).

Cloudflare's [Developer Platform](https://developers.cloudflare.com/) lets you develop full-stack applications with access to resources such as storage and AI, all deployed to a global edge network. This adapter builds your Vite project for deployment through Cloudflare.

### Manual install

The following steps assume you already have a Vite project.

<Steps>

1. Add `@vite-deploy/cloudflare` and `wrangler` to your project's dependencies using your preferred package manager:

    <PackageManagerTabs>
      <Fragment slot="npm">
        ```sh
        npm install @vite-deploy/cloudflare wrangler
        ```
      </Fragment>
      <Fragment slot="pnpm">
        ```sh
        pnpm add @vite-deploy/cloudflare wrangler
        ```
      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn add @vite-deploy/cloudflare wrangler
        ```
      </Fragment>
    </PackageManagerTabs>

2. Add the plugin to your Vite config file:

    ```js title="vite.config.js" ins={1,5-7}
    import cloudflare from "@vite-deploy/cloudflare";
    import { defineConfig } from "vite";

    export default defineConfig({
      plugins: [cloudflare({
        output: "server"
      })]
    });
    ```

3. Add a NPM script to generate types:

    ```jsonc title="package.json" ins={4}
    {
      "scripts": {
        // ...
        "cf-typegen": "wrangler types"
      }
    }
    ```

4. Update your TypeScript config file to include necessary types:

    ```jsonc title="tsconfig.json" ins={5,10}
    {
      "compilerOptions": {
        "types": [
          "vite/client",
          "@vite-deploy/cloudflare/types"
        ]
      },
      "include": [
        "src",
        "worker-configuration.d.ts"
      ]
    }
    ```

5. Create a `wrangler.jsonc` config file:

    ```jsonc title="wrangler.jsonc"
    {
      "$schema": "./node_modules/wrangler/config-schema.json",
      "name": "your-site",
      // Set this to today's date
      "compatibility_date": "2026-03-17",
      "main": "./src/worker.ts",
    }
    ```

6. Create the worker entrypoint:

    ```ts title="src/worker.ts"
    export default {
      fetch(request) {
        const url = new URL(request.url);
        return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
      },
    } satisfies ExportedHandler<Env>;
    ```

7. Start the development server:

    <PackageManagerTabs>
      <Fragment slot="npm">
        ```sh
        npm run dev
        ```
      </Fragment>
      <Fragment slot="pnpm">
        ```sh
        pnpm dev
        ```
      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn dev
        ```
      </Fragment>
    </PackageManagerTabs>

</Steps>

## Options

The Cloudflare adapter accepts [output related options](/outputs/). It also accepts the following:

### `config`

<p>
**Type:** `PluginConfig | undefined`<br />
**Default:** `undefined`
</p>

Options forwarded to [`@cloudflare/vite-plugin`](https://developers.cloudflare.com/workers/vite-plugin/), except `viteEnvironment`:

```js title="vite.config.js" {7-9}
import cloudflare from "@vite-deploy/cloudflare";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [cloudflare({
    // ...
    config: {
      inspectorPort: 9230
    }
  })]
});
```

## Worker entrypoint

To handle requests, you need a worker. The `main` field in your wrangler config file specifies what module should be used. It accepts:

- Paths relative to Vite's root: `./src/worker.ts`.
- Absolute paths: `/foo/worker.ts`.
- Package specifiers: `@my-pkg/worker`.

```jsonc title="wrangler.jsonc" {6}
{
  "$schema": "./node_modules/wrangler/config-schema.json",
  "name": "your-site",
  // Set this to today's date
  "compatibility_date": "2026-03-17",
  "main": "./src/worker.ts",
}
```

You then need to implement the worker:

```ts title="src/worker.ts"
export default {
  fetch(request, env, ctx) {
    const url = new URL(request.url);
    return new Response(`Running ${url.pathname} in ${navigator.userAgent}!`);
  },
} satisfies ExportedHandler<Env>;
```

## Node.js compatibility

Cloudflare Workers support most Node.js runtime APIs through the `nodejs_compat` compatibility flag. This includes commonly used modules like `node:buffer`, `node:crypto`, `node:path`, and many others. See the [full list of supported Node.js APIs](https://developers.cloudflare.com/workers/runtime-apis/nodejs) in Cloudflare's documentation.

To enable Node.js compatibility, add the `nodejs_compat` flag to your Wrangler configuration:

```jsonc title="wrangler.jsonc"
{
  "compatibility_flags": ["nodejs_compat"],
}
```

Then use the `node:*` import syntax in your server-side code:

```js
import { Buffer } from 'node:buffer';
```

For Node.js APIs not yet supported in the Workers runtime, Wrangler can inject polyfills (requires `nodejs_compat` and a compatibility date of 2024-09-23 or later).

## Static assets

Static assets work out of the box. The adapter injects a header to cache assets under [`build.assetsDir`](https://vite.dev/config/build-options#build-assetsdir).
